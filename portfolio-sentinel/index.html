About this prototype

  
Rule-based and fully client-side; nothing on this page is transmitted anywhere. It demonstrates the validation-agent pattern on a mixed portfolio (multifamily, hotel, industrial/flex): every record checked, clean data passing silently, and only exceptions reaching a person, with evidence and a suggested correction attached. The 1,284 records and three properties are synthetic.

  
    
    
  • Where this runs in productionReads from Yardi Voyager and RealPage through governed connectors into Microsoft Fabric / OneLake; orchestrated with Power Automate or Logic Apps and MCP tool servers; agents composed in Claude Code and Copilot Studio.
    • 
    
  • Governed by designEvery deployed agent documented: purpose, owners, data accessed, prompts, evaluation results, and known limitations, with read/write boundaries enforced at the connector, not the prompt.
    • 
    
  • Measured, not demo-readyEvaluation pipeline scores the checks against labeled cases before deploy; pass rates, exception aging, and hours returned to the accounting team tracked in Power BI.
